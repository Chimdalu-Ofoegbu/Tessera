<objective>
Define the metric contract that makes a bare or unsourced number impossible by type, and the normalized entity shapes every downstream layer obeys. The centerpiece is `Metric<T>` — a discriminated union `{ ok: true, value: T, provenance } | { ok: false, insufficient: true, provenance }` — so (a) a value can never exist without provenance and (b) `INSUFFICIENT_DATA` is a first-class outcome, not a magic `0`/`null`. All shapes are zod schemas with `z.infer` types (single source of truth), mirroring the real Renaiss API fields so mock and real sources are honest. (DATA-03, PROV-02.)

Output: `src/data/schema.ts`, `src/data/metric.ts`, `src/core/thresholds.ts`, and passing Vitest tests proving the envelope rejects unsourced values and represents insufficiency.
</objective>

| Boundary | Description |
|----------|-------------|
| external data -> normalized entities | All inbound data must pass zod validation before entering the app |
| metric value -> UI/API | A metric must carry provenance or be explicitly insufficient |

| T-02-01 | Tampering / Information Integrity | Unsourced or fabricated values | mitigate | `Metric<T>` discriminated union makes a value without `provenance` unrepresentable; `metricSchema` rejects `{ ok:true, value }` lacking provenance (test-asserted). |
| T-02-02 | Spoofing (data provenance) | Source attribution | mitigate | `Provenance.source` is a closed `SourceId` enum (`seed`/`renaiss`); every metric names its origin. |
