Endpoints (see RENAISS-API.md): getCategories←`GET /v1/indices`, getCategory←`GET /v1/indices/{game}`, getIndexSeries←index sparkline or a representative card `/series`, getRecentSales←`GET /v1/trades/recent`, getFeaturedMovers←`GET /v1/cards/featured`, health←`GET /v1/health`.

- Do NOT hit the live API in tests. Capture a representative response into `fixtures/indices.sample.json` (hand-authored from the documented schema) and test the pure mappers against it.
</notes>

## Trust Boundaries
| Boundary | Description |
|----------|-------------|
| app -> external Renaiss API | Outbound requests to a fixed third-party host with optional credentials |
| API response -> normalized entities | Untrusted external JSON must be validated before use |
| env -> credentials | API key/secret must come only from env, never logged or committed |

## STRIDE Threat Register
| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-01 | Server-Side Request Forgery | Renaiss client URL | mitigate | Base URL is a fixed allowlisted constant; only fixed path segments/enum ids are appended — no user-controlled host or scheme. |
| T-04-02 | Information Disclosure | API key/secret | mitigate | Credentials read from env only, attached as headers, never logged, never committed (`.env` gitignored per 01-01). |
| T-04-03 | Denial of Service | Rate limit (10/day anon) | mitigate | In-memory TTL cache + `429`/`Retry-After` handling → return cached/insufficient, not a crash; MockSource is the default so the demo never depends on live quota. |
| T-04-04 | Tampering | Malformed API payloads | mitigate | Responses mapped through pure functions and validated against zod schemas; failures degrade to insufficient. |
</threat_model>
