# Quick Task 260708-kg5: Strict light mode + remove theme toggle

## Goal
The terminal UI (overview + detail + watchlist drawer + nav) always renders LIGHT, with no way to
switch to dark and no theme toggle control. Home hero + its dark footer band = landing design, untouched.

## Tasks
1. **src/App.tsx** — pin `theme` to `'light'` (drop the `useState`/localStorage `tesseraTheme` read+write);
   ui object `dark: false, theme: 'light'`, remove `toggleTheme`; remove now-unused `readLS` helper; drop
   `theme` from the useMemo deps. All `data-tt-theme={ui.theme}` → "light"; all `ui.dark` branches → light.
2. **src/ui.ts** — remove `toggleTheme: () => void` from the `UI` interface (keep `dark`, `theme`).
3. **src/components/Nav.tsx** — delete the theme toggle `<button className="btn-ghost" onClick={ui.toggleTheme}>`
   (☀/☾ + `.nav-btn-label`); move `marginLeft:'auto'` onto the Watchlist button so it stays right-aligned.
4. **src/index.css** — remove the now-unused `.nav-btn-label` rule (in the ≤620px block). Keep `.btn-ghost`
   (still used by the drawer ESC button). Responsive nav media queries otherwise untouched.
5. **src/components/WatchlistDrawer.tsx** — convert hardcoded dark → light. Add `data-tt-theme={ui.theme}`
   to the outer fixed container; panel bg `var(--t-bg)`, rows `var(--t-panel)`+`var(--t-hair)` border,
   text `var(--t-ink)`/`var(--t-ink2)`/`var(--t-ink3)`, gold `var(--t-gold)`, borders `var(--t-hair2)`;
   `changeColor(...,true)` → `changeColor(..., ui.dark)`. Keep the `rgba(0,0,0,.55)` scrim.

## Do NOT touch
Hero.tsx, Footer.tsx (landing), the responsive nav breakpoints (960/620), the risk/index engines, data, API.

## Verify
- preview (dev, 5173): overview + scored detail (NBA) + open watchlist drawer → LIGHT bg/dark ink,
  NO nav theme toggle, drawer light w/ readable contrast, no console errors. Screenshot overview/detail/drawer.
- `pnpm typecheck` + `pnpm build` green (no dangling `toggleTheme`/`readLS` refs). No desktop/responsive regression.
