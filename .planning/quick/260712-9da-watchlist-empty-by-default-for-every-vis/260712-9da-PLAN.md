# Quick Task 260712-9da: Watchlist empty by default + session-only

## Requirements (user)
1. The watchlist starts **empty for every visitor** until they add an index themselves.
2. It is cleared **only** on hard refresh and/or clearing cookies — i.e. it must not
   persist beyond the browser page session.

## Findings (current behavior — both wrong for the ask)
`src/App.tsx`:
- `readWatch()` seeded new visitors with `['one-piece', 'sports']` when nothing was
  stored (line 18) — so the watchlist was never empty, and `sports` isn't even a live
  category anymore.
- A `useEffect` mirrored `watch` into `localStorage['tesseraWatch']` — which survives
  hard refresh, tab close, and browser restart. The opposite of the requested lifetime.

## Decision — in-memory (React state), no web storage
Browsers do **not** expose hard-vs-soft reload to JS, and no web storage clears on a
hard refresh specifically (localStorage and sessionStorage both survive reloads). The
only mechanism that satisfies "clear on hard refresh" is in-memory state. Because the
app is a SPA (screens switch via React state, never a page load), in-memory state:
- starts empty for every visitor / tab,
- survives all in-app navigation (Overview ↔ Detail ↔ drawer, opening/closing it),
- resets to empty on any full page reload (soft or hard) and on cookie/site-data clear.
sessionStorage was rejected: it would survive a hard refresh, failing requirement 2.

## Changes
1. `src/App.tsx` — delete `readWatch()`; init `useState<string[]>([])`; delete the
   `localStorage.setItem` effect. `toggleWatch` (add/remove) is unchanged — still pure
   in-session state. Comment documents the session-only lifetime + the hard/soft caveat.
2. `src/components/WatchlistDrawer.tsx` — label `LOCAL ONLY` → `SESSION ONLY` (the
   drawer already ships a clean empty state: "Nothing pinned yet…"). Honesty in the UI.
