# Build always ships live data (never a hand-frozen snapshot)

The prod `public/api/*.json` snapshot was generated once by hand and froze at asOf Jul 7
while the live index moved on. Now every **production build regenerates it from live
Renaiss**, so a deploy can no longer ship stale data.

## What shipped
- **`scripts/gen-snapshot.mjs`** ‚Äî loads the real compute layer via Vite `ssrLoadModule`
  (same mechanism as the dev middleware) and runs `buildOverview` /
  `buildCategoryAnalytics` / `source.health()` over the live `RenaissSource`, writing all
  six `public/api/*.json` files. Reusing the exact handler compute path guarantees shape
  parity with what the frontend fetches (the client does no schema validation ‚Äî just
  `fetch` + cast ‚Äî so shape correctness is on us, and this delivers it).
  - **Gated:** live only on `--live` / `USE_RENAISS=1` / `VERCEL_ENV=production` / partner
    creds. Local `pnpm build` no-ops ‚Üí never spends the public tier's 10 req/day.
  - **All-or-nothing + fail-safe:** writes only when the full set fetched and validated as
    live (`source==='renaiss'`, non-empty); any rate-limit/outage/timeout keeps the
    committed snapshot and `exit 0` ‚Üí the build never breaks.
  - **Prunes** category/index files for games Renaiss no longer lists.
- **`package.json`** ‚Äî `build` = `node scripts/gen-snapshot.mjs && vite build`; added
  `gen:snapshot` (`--live`) for manual refresh.
- **`public/api/*.json`** ‚Äî regenerated to live Jul-14 (fresh fallback floor, was Jul-7).
- **README** ‚Äî documents the automated refresh, fail-safe, and local skip.

## Freshness envelope (honest)
Every **deploy** ships live data. Between deploys the snapshot is static ‚Äî the public tier
(10 req/day) rules out per-request live fetching. Truly hands-off "never stale while idle"
needs a scheduled rebuild (daily GitHub Action / Vercel cron + deploy hook) or Renaiss
partner credentials (10k/day) for runtime-live. Left as a follow-up for the user to approve
(standing automation / credentials ‚Äî not wired unprompted).
