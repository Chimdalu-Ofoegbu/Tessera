# Quick Task 260712-7db: Add Tessera product favicon (replace tab placeholder)

## Problem
The browser tab shows the generic placeholder page icon: `index.html` declares **no
favicon** (head is `<meta charset>`, `<meta viewport>`, `<title>` only) and `public/`
holds only API JSON — there is no icon asset anywhere. Title reads
"Tessera — Collector Market Intelligence" (matches the truncated tab in the screenshot).

## Source of truth for the mark
The brand mark already lives in code — `src/components/Nav.tsx` `Logo()` (lines 9-14):
a 2×2 CSS grid of 7px tiles, `gap:2`, `transform: rotate(45deg)` → a mosaic-tile diamond
(literally a *tessera*). Home-screen colors, clockwise from top-left cell:
gold `#C9A961` · green `#2E8065` · dark-gold `#8F6F26` · ink `#1B1710`.
Same grid + same 45° rotation ⇒ favicon reads identical to the live nav logo.

## Task 1 — create `public/favicon.svg`
- 32×32 viewBox, scalable (viewBox only, no fixed px) so it stays crisp at 16/32/180px.
- Cream rounded-square plaque `#F2EDE3` (the site's paper) with a hairline
  `#1B1710 @ .10` edge — gives the ink tile contrast and defines the plaque on both
  dark and light tab strips (a transparent mark would lose the ink tile on dark tabs).
- 4 tiles (size 7.5, gap 2, block 17 centered at 16) inside `rotate(45 16 16)`, colors
  exactly as above. Diagonal ≈24 < 32 ⇒ ~4px margin, no clipping.

## Task 2 — wire it into `index.html`
- `<link rel="icon" type="image/svg+xml" href="/favicon.svg" />` (Vite serves `public/`
  at root). Modern evergreen browsers (the demo target) render SVG favicons.
- `<meta name="theme-color" content="#F2EDE3" />` — matches the strict-light paper;
  tints mobile browser chrome. Natural companion, on-brand.

## Verify
- `favicon.svg` is well-formed XML; `/favicon.svg` resolves.
- Render the SVG at 16/32/64px on light + dark chips (playwright-core, as used for the
  hero still) to confirm all four tiles read on either tab background.
- `pnpm build` green (index.html is the Vite entry; a malformed head would break it).
